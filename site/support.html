Common Issues

Spoofing isn't working after I installed the extension

The extension injects into pages at load time, so any tabs that were already open when you installed or enabled it won't be protected yet.

Fix: Refresh every tab you want to protect after enabling Location Protection. If it still doesn't work, try disabling and re-enabling the extension, then refresh again.

VPN Sync shows a timeout or network error

VPN Sync works by calling a few public IP geolocation services to detect your VPN exit region. Some VPNs or firewalls block outbound requests to these services, which causes timeouts.

Things to try:

  • Temporarily disable your VPN's firewall or "kill switch" and try again
  • Check if your VPN has a setting that blocks third-party API calls
  • If the issue persists, use the Search City or Enter Coordinates tabs to set your location manually instead

VPN Sync is a convenience feature — manual location entry always works regardless of network restrictions.

Spoofing stopped working after a browser update

Browser updates occasionally change how extensions interact with page APIs. We track these changes and release fixes as quickly as possible.

Fix: Make sure you're on the latest version of GeoSpoof. Check the extension's version in the Details tab of the popup and compare it to the latest release on GitHub. If you're behind, update through your browser's extension manager.

A specific website isn't being spoofed

A small number of sites use server-side location detection (based on your IP address) rather than the browser Geolocation API. GeoSpoof overrides browser APIs only — it does not change your IP address.

For full location consistency, use GeoSpoof alongside a VPN pointed at the same region. The Sync with VPN feature makes this easy.

Some sites also run location checks inside Web Workers, which extensions cannot inject into. This is a known browser limitation.

The extension works on desktop but not on my phone

On Firefox for Android, the extension is fully supported on Firefox 140 and later. Make sure your Firefox is up to date.

On iOS and macOS Safari, the extension is available through the App Store. Safari extensions require explicit per-site permission — tap the puzzle piece icon in the address bar and enable GeoSpoof for the site you want to protect.

Chrome for iOS and Android does not support extensions — this is a Google limitation, not something we can work around.

WebRTC Protection isn't available / greyed out

WebRTC Protection uses a browser privacy API that is not available on all platforms. It is supported on Firefox and Chromium-based browsers (Chrome, Brave, Edge) on desktop.

It is not available on Safari or Firefox for Android due to platform API limitations.

I see "Extensions cannot run on this page"

Browsers restrict extensions from running on certain built-in pages such as about:blank, chrome://, about:newtab, the browser's own settings pages, and extension store pages. This is a browser security boundary that cannot be bypassed.

GeoSpoof works on all normal websites. If you're seeing this notice on a regular site, please open an issue with the URL.
